## 🎯 Sobre este Manual

Este é um **manual completo e intuitivo** sobre Git e GitHub, criado especialmente para **estudantes de programação iniciantes** que nunca tiveram contato com essas ferramentas.

O material foi desenvolvido com uma abordagem **didática, visual e prática**, incluindo:
- ✅ Explicações claras e detalhadas
- ✅ Interface colorida e atrativa
- ✅ Exemplos práticos
- ✅ Passo a passo ilustrado
- ✅ Comandos fundamentais
- ✅ Solução de problemas comuns
- ✅ Exercícios para fixação

## 📖 O que você vai aprender

### 1. **Conceitos Fundamentais**
- O que é Git e GitHub
- Diferença entre Git e GitHub
- Sistema de controle de versão distribuído

### 2. **Primeiros Passos**
- Criar conta no GitHub
- Instalar Git no computador
- Configurar Git Bash
- Instalar GitHub Desktop

### 3. **Configuração**
- Configuração inicial do Git
- **Geração e configuração de chave SSH** (seção aprofundada)
- Autenticação segura

### 4. **Trabalhando com Repositórios**
- Criar repositórios no GitHub
- Clonar repositórios
- Fluxo de trabalho básico

- `git status`, `git add`, `git commit`
- `git push`, `git pull`
- `git log`, `git diff`
- `git branch`, `git merge`
- E muito mais!

### 6. **Integração com VS Code**

- Branches e merges

- Arquivo .gitignore
- Boas práticas
- Comandos de emergência

## 🚀 Como usar este manual

### Visualização Online
Basta abrir o arquivo `manual_github.html` no seu navegador preferido para ter acesso ao manual completo e interativo.

### Download
1. Clone este repositório:
```bash
git clone https://github.com/seu-usuario/manual-git-github.git
```

2. Abra o arquivo `manual_github.html` em qualquer navegador

### Ou baixe diretamente
- Clique no arquivo `manual_github.html`
- Clique em "Raw" ou "Download"
- Salve no seu computador

## 🤝 Como Contribuir

Contribuições são bem-vindas! Se você encontrou algum erro, tem sugestões de melhorias ou quer adicionar conteúdo:

1. Faça um fork do projeto
2. Crie uma branch para sua feature (`git checkout -b feature/MinhaContribuicao`)
3. Commit suas mudanças (`git commit -m 'Adiciona nova seção sobre...'`)
4. Push para a branch (`git push origin feature/MinhaContribuicao`)
5. Abra um Pull Request
